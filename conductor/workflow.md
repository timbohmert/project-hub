# Workflow

## Orchestration
- **Conductor:** Used to manage the project roadmap and individual workstreams (tracks).
- **Tracks:** Each major feature or service will be isolated in its own track to maintain modularity and allow for independent technological experimentation.

## Principles
- **Vibe Coding for UI:** Rapid prototyping of the frontend to establish look and feel.
- **Rigorous Engineering for Services:** Following best practices and potentially over-engineering for learning.
- **Modular Evolution:** Ensuring clear API boundaries to allow swapping or adding microservices easily.
